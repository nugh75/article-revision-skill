# Accepted Anglicisms — Italian Articles

Whitelist of English terms tolerated in Italian academic prose because they are established conventions or lack a precise equivalent. Each term should be italicized at first occurrence unless journal norms require otherwise.

## Technology / AI

- *artificial intelligence* (AI), *generative AI* (GenAI)
- *bias*
- *prompt*, *prompt engineering*
- *output*, *input*
- *feedback*
- *framework*
- *dataset*, *cluster*
- *machine learning* (ML), *deep learning*
- *large language model* (LLM)
- *chatbot*, *chat-bot*
- *e-learning*

## Education and Research

- *cheating*
- *ghostwriting*
- *scaffolding*
- *paywall*
- *peer review*, *peer reviewer*
- *abstract*
- *keyword*, *keywords*
- *open access*
- *workshop*
- *brainframe* (de Kerckhove; technical term, always italicized)

## Statistics and Methodology

- *survey*
- *coding*, *codebook*
- *trial*
- *benchmark*
- *baseline*
- *outlier*

## Prefer Italian Equivalents

| English term to avoid | Preferred Italian equivalent |
|---|---|
| performance | prestazione |
| trend | tendenza |
| tool (generic sense) | strumento |
| look | aspetto, immagine |
| skill (singular) | competenza, capacità |
| training (generic sense) | formazione |
| approach | approccio |
| budget (generic sense) | bilancio, vincolo |
| target | obiettivo, destinatario |
| device | dispositivo |
| location | sede, contesto |
| step | passo, fase |
| issue | problema, questione |
| paper (scientific article) | articolo |

## Operational Notes

- In `ARTICLE_LANG=it`, any proposal that introduces an anglicism not in this whitelist must flag it in chat: `The anglicism <term> is not in the whitelist. Use the Italian equivalent <...>?`
- The user may add terms permanently to the whitelist by confirming in chat. The skill updates this file, but does not commit unless explicitly asked.
