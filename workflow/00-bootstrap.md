# 00 — Bootstrap

Run **first** when the skill is invoked. This step is idempotent: if all artifacts are already in place, it exits quickly and proceeds to `10-setup.md`.

The bootstrap establishes the revision environment: Python venv with required packages, `.bib` file, `.env` with editorial parameters and credentials, and editorial norms file. Each creation step asks the user first; never write files silently.

## Step A — Python Virtual Environment

1. Look for a venv at `<project-root>/.venv/`.
2. If absent, ask:

   > I cannot find a venv for the skill scripts. Do you want me to create one in `.venv/` and install packages from `requirements.txt`?
   > (yes / no / specify alternative path)

3. On confirmation:

   ```bash
   python3 -m venv .venv
   .venv/bin/python -m pip install --upgrade pip
   .venv/bin/python -m pip install -r .claude/skills/article-revision/requirements.txt
   ```

4. If the user refuses venv creation, ask before falling back:

   > Do you want to use system Python for this session only? Some scripts may fail if packages are missing.
   > (yes / no)

   If the answer is `no`, skip Python-based checks and note that scripts are unavailable until a venv or interpreter is configured.

5. If a venv already exists, verify the packages match `requirements.txt`:

   ```bash
   .venv/bin/python -m pip check
   .venv/bin/python -m pip install -r .claude/skills/article-revision/requirements.txt --upgrade-strategy only-if-needed
   ```

6. Remember the interpreter for the session:

   - venv accepted/found: `PYTHON_BIN=.venv/bin/python`
   - alternative path accepted: `PYTHON_BIN=<custom-path>/bin/python` or the explicit interpreter path
   - system Python accepted: `PYTHON_BIN=$(command -v python3)`

7. Ask before persisting `PYTHON_BIN` to `.env`:

   > Do you want to save `PYTHON_BIN=<path>` in `.env` for future sessions?
   > (yes / no)

8. All subsequent Python scripts in this workflow must run with `PYTHON_BIN`. Never call `python3 scripts/...` directly unless the user explicitly chose system Python.

## Step B — `.bib` File

1. Read `BIBLIOGRAPHY_BIB_PATH` from `.env`. If `.env` does not exist yet, default to `bibliography/reference.bib`.
2. If the path does not exist, ask:

   > I cannot find `<path>/reference.bib`. Do you want me to create an empty one there?
   > (yes / specify alternative path)

3. On confirmation, create the directory if needed and write a minimal `.bib`:

   ```bibtex
   % Bibliography file generated by article-revision skill on YYYY-MM-DD
   % Add entries below using the bibliography skill, if available.
   ```

## Step C — `.env` File

1. Look for a `.env` at the project root.
2. If absent or missing required keys, walk through the keys interactively. For each missing/empty key, ask one question or batch them.

Required:

- `EDITORIAL_LIMIT_CHARS` (or `EDITORIAL_LIMIT_WORDS`)
- `EDITORIAL_NORMS_PATH`
- `BIBLIOGRAPHY_BIB_PATH`

Optional but recommended:

- `CROSSREF_USER_AGENT` and `OPENALEX_USER_AGENT` (form: `ProjectName/1.0 (mailto:author@example.org)`)
- `ZOTERO_USER_ID`, `ZOTERO_API_KEY`, `ZOTERO_GROUP_ID`
- `ARTICLE_LANG` (usually empty; auto-detected)
- `PYTHON_BIN` (usually `.venv/bin/python`)
- `AUTO_BUMP_THRESHOLD` (default `5`)

3. Append new keys to `.env` or create the file. Never overwrite existing values without asking.
4. Re-load `.env` after edits and continue.

### Efficient Prompt

If `.env` is missing several keys, batch the questions:

```text
To complete the environment I need these values. Reply by number, or leave blank to skip:
1. Character limit (default 40000):
2. Editorial norms path:
3. Bibliography path (default bibliography/reference.bib):
4. Crossref/OpenAlex User-Agent (format `Project/1.0 (mailto:you@host)`):
5. Zotero — user_id, api_key, group_id (comma-separated):
6. Python interpreter (default `.venv/bin/python`):
7. Auto-bump threshold (default 5):
```

## Step D — Editorial Norms File

1. After `EDITORIAL_NORMS_PATH` is set, check whether the file exists.
2. If absent, ask:

   > The editorial norms file (`<path>`) does not exist. Do you want to:
   > - **(a)** create an empty placeholder
   > - **(b)** copy it from another project (provide source path)
   > - **(c)** postpone and proceed without norms (the skill will work but cannot enforce journal rules)

3. Apply the choice. For (a):

   ```markdown
   # Editorial Norms — <journal>

   <!-- Add binding journal rules here:
        - maximum length
        - citation style
        - quotation marks, italics, bold
        - section structure
        - bibliography format
   -->
   ```

## Step E — Project Structure

Verify or create:

```text
<project-root>/
├── articles/                       # if absent, create
└── revisions/                      # if absent, create
```

Do not move existing files into these folders without asking.

After all bootstrap steps, output one chat block:

```text

- venv: <.venv/ | system>
- python: <PYTHON_BIN | unavailable>
- bibliography: <path> (X entries)
- editorial norms: <path>
- .env: <path> (M keys configured)
- folder structure: <ok | created>

Proceed with `10-setup.md` to identify the active article? (yes / no)
```

If everything was already in place, this block can be condensed to: `Environment already configured. Proceeding with setup.`
